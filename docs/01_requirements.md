# TypeLab — 需求文档

## 背景

见 `docs/00_proposal.md`。

## 用户

| 角色 | 描述 |
|---|---|
| 练习者 | 登录后选择课程、打字、查看成绩、上排行榜 |
| 访客 | 未登录，可以打字但成绩不保存、不上榜 |
| 管理员 | 通过 PR 维护 `src/lessons/` 题库内容 |

## 功能需求

### F01 — 用户认证
- 用户可以用邮箱+密码注册账号
- 用户可以登录和退出
- 登录状态在刷新后保持（开发阶段由 mock session 模拟；上线阶段由 Supabase session 自动处理）
- 未登录用户可以使用打字功能，但成绩不保存

### F02 — 课程选择
- 用户可以浏览所有课程，按类别筛选（warmup / 排序 / 树 / DP / 图 / JS）
- 每条课程显示：标题、难度（1-5星）、类别标签、预估完成时长
- 用户可以点击任意课程进入打字界面

### F03 — 打字引擎（核心）
- 上方显示目标文本，下方为输入区
- 实时逐字符对比：正确字符高亮绿色，错误字符高亮红色
- 显示当前实时 WPM（每5秒更新一次）
- 显示当前准确率
- 打完最后一个字符自动结束，跳转结果页
- 支持 Tab 键输入制表符（代码缩进必须）
- 不支持退格修正错误（强迫专注，可在后期作为可选项）

### F04 — 成绩结果
- 显示本次 WPM、准确率、用时、错误字符数
- 显示本课程的历史最佳成绩（如已登录）
- 提供"再来一次"和"选其他课程"两个按钮
- 已登录用户成绩自动提交保存

### F05 — 排行榜
- 显示所有用户的最高 WPM 记录（每人只取最佳一条）
- 显示字段：排名、用户名、WPM、准确率、所用课程
- 每30秒自动刷新

### F06 — 个人主页
- 显示该用户所有历史成绩，按时间倒序
- 显示每个类别的最高 WPM（折线图展示进步趋势，用 ECharts）
- 仅登录用户可见自己的数据

### F07 — 题库（管理员维护）
- 题库以 JSON 文件存储在 `src/lessons/` 目录
- 管理员通过提交 PR 增删改课程内容
- 每条课程包含：id、title、category、difficulty、text、note

## 非功能需求

- **性能**：打字引擎输入延迟 < 16ms（不卡键）
- **兼容性**：支持 Chrome 110+、Edge 110+，不要求 Firefox/Safari
- **部署**：Vercel 自动部署，每个 PR 生成预览链接
- **成本**：Vercel 免费 + Supabase 免费 tier，月费用为零

## 输入与输出

| 输入 | 输出 |
|---|---|
| 用户键盘输入 | 实时字符状态（正确/错误/未到达） |
| 课程 JSON 文件 | 渲染的打字文本 |
| 完成信号 | WPM、准确率、错误数 |
| 已登录用户完成 | 通过 `src/lib/db.js` 保存成绩（开发阶段写入 mock 数据，上线阶段写入 Supabase results 表） |
| 排行榜请求 | 通过 `src/lib/db.js` 读取排行榜数据（开发阶段 mock 聚合，上线阶段 Supabase 聚合） |

## 数据模型草稿

开发阶段使用 `src/lib/db.js` 中的 mock 数据结构模拟；上线阶段使用 Supabase Auth 和 results 表。

```
users        由 Supabase Auth 管理，无需手动建表
             字段：id (uuid), email, created_at

results
  id          uuid, primary key
  user_id     uuid, 关联 auth.users.id
  lesson_id   text, 对应 JSON 文件中的 id 字段
  wpm         integer
  accuracy    numeric(5,2)
  duration    integer  (秒)
  created_at  timestamptz, default now()

leaderboard_view  (数据库视图，非物理表)
  SELECT user_id, max(wpm) as best_wpm, ...
  FROM results GROUP BY user_id
```

## 边界情况

- 用户打到一半刷新页面：成绩丢失，重新开始，不保存不完整记录
- 课程文本包含换行符：打字引擎需要正确渲染和处理 `\n`
- 网络提交成绩失败：显示提示"成绩保存失败，请检查网络"，本地仍显示结果
- 同一用户同一课程多次完成：全部保存，排行榜只取最高分

## 验收标准

| 编号 | 标准 |
|---|---|
| AC01 | 未登录用户可以完成一次打字，结果页正常显示，但无保存提示 |
| AC02 | 注册后登录，完成打字，刷新排行榜能看到自己的名字和成绩 |
| AC03 | 打字过程中 WPM 数字实时变化，字符颜色实时反馈 |
| AC04 | Tab 键在打字界面中输入制表符而非切换焦点 |
| AC05 | 在 `src/lessons/` 新增一个 JSON 文件，刷新课程列表页后能看到该课程 |
| AC06 | 个人主页能看到历史所有成绩，最新的在最上面 |
